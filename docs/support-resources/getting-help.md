# Getting Help

How to get support for Third Space Streaming Platform.

## Support Channels by Tier

### Free Tier

**Community Discord**
- Join: [discord.gg/8pVjBjCe](https://discord.gg/8pVjBjCe)
- Response time: Best effort from community and team
- Best for: General questions, troubleshooting, feature discussions
- Channels: #general, #troubleshooting, #feature-requests

### Plus Tier

**Email Support**
- Email: support@thirdspaceinteractive.ca
- Response time: Within 48 hours (business days)
- Include: Project ID, error details, steps to reproduce
- Plus: Access to Community Discord

### Pro Tier

**Priority Email Support**
- Email: support@thirdspaceinteractive.ca
- Response time: Within 24 hours (includes weekends)
- Direct connection to engineering team
- Plus: All Plus tier support options

### Enterprise Tier

**Dedicated Support Channels**
- **Slack:** Private shared channel with Third Space team
- **Email:** Dedicated support alias
- **Phone:** Scheduled calls with engineering
- **Response time:** Within 4 business hours
- **Account manager:** Dedicated contact
- Plus: All lower tier support options

## Before You Contact Support

### Gather Information

Help us help you faster by providing:

1. **Project ID** - Found in dashboard URL or project settings
2. **Error messages** - Full text or screenshot
3. **Steps to reproduce** - What were you doing when issue occurred?
4. **Browser & OS** - Chrome 102 on Windows 11, Safari on iOS 16, etc.
5. **Network info** - Corporate network, home wifi, mobile hotspot?
6. **Screenshots** - Console errors, deployment status, etc.

### Check Documentation First

Many issues are covered in our docs:
- [FAQ](../faq) - Common questions
- [Troubleshooting](../troubleshooting) - Issue-specific guides
- [Upload Issues](../troubleshooting/upload-issues) - Upload troubleshooting
- [Deployment Issues](../troubleshooting/deployment-issues) - Deployment errors

### Search Discord

Someone may have already asked your question:
- Use Discord search (top right)
- Check pinned messages in channels
- Browse #troubleshooting channel

## How to Report Issues

### Email Support Template

```
Subject: [Project ID] Brief description of issue

Project ID: proj_abc123xyz
Tier: [Free/Plus/Pro/Enterprise]
Issue: [Brief description]

What I was trying to do:
[Describe your goal]

What happened instead:
[Describe the problem]

Steps to reproduce:
1. [First step]
2. [Second step]
3. [Issue occurs]

Environment:
- Browser: Chrome 120 on Windows 11
- Network: Home wifi, 100 Mbps
- Package: Linux build, 3.5GB

Additional context:
[Any other relevant details]

Screenshots/logs:
[Attach or link]
```

### Discord Support Format

```
**Issue:** Stream won't deploy
**Project ID:** proj_abc123xyz
**Tier:** Pro
**What I tried:** Re-uploaded package twice
**Error:** ERR_002 - Missing executable
**Screenshot:** [link]
```

## Escalation Process

### If Issue Isn't Resolved

1. **Reply to original email** - Don't start new thread
2. **Ask for escalation** - "Can this be escalated?"
3. **Provide updates** - Any new info or tests you've tried

### Critical Issues (Enterprise)

For production-down situations:
- Use Slack channel (immediate notification)
- Mark as URGENT in subject
- Follow up with phone call if no response in 30 min

## Bug Reports

### How to Report Bugs

Use the same channels as support, but clearly mark as bug:

**Subject:** [BUG] Brief description

Include:
- Expected behavior
- Actual behavior
- Steps to reproduce consistently
- Browser console logs (if applicable)

### Bug Bounty (Enterprise)

Enterprise tier includes bug bounty program:
- Security vulnerabilities: Up to $500
- Critical bugs: Up to $200
- Minor bugs: Recognition in changelog

Contact security@thirdspaceinteractive.ca

## Emergency Support

### What Qualifies as Emergency?

**Enterprise tier only:**
- Production system completely down
- Security breach or data leak
- Affecting 50+ end users
- Time-sensitive event (trade show, presentation)

### How to Reach Emergency Support

1. **Slack:** Post in dedicated channel with @oncall
2. **Phone:** Call enterprise support line (provided in onboarding)
3. **Email:** Use URGENT flag

### Response Time

- Acknowledgment: Within 30 minutes
- Engagement: Within 2 hours
- Resolution: Best effort, typically within 24 hours
